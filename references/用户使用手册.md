# 深耕 · 销售管理 v4.1 用户使用手册

## 快速开始

v4.1 是完整版，包含从 v1.0 到 v4.1 的全部 22+ 项能力。

### 第一步：首次启动

助理会逐步引导您完成 4 步初始化：

1. **角色选择**：一线销售 / 销售团队负责人（选择"负责人"将激活 18 项团队管理功能）
2. **产品类型**：软件/SaaS、工业设备、专业服务、其他
3. **方法论偏好**：SPIN（默认）/ BANT / MEDDIC
4. **数据导入**：上传 Excel 名单 / 从零开始

完成后，您的销售工作台即刻就绪。

### 第二步：添加第一个客户

您可以用以下任意方式开始：

**方式一：直接添加**
> "添加客户：苏州云创科技，电子元器件制造业，300人规模"

**方式二：批量挖掘**
> "帮我找苏州的制造业企业，营收5000万以上"

**方式三：导入名单**
> 上传一份 Excel/CSV 文件，助理会自动解析并补全信息

## 功能详解

### 一、客户挖掘与信息补全（v1.0，v4.1 Agent-Reach增强）

**触发方式**：描述您的目标客户条件

**示例对话**：
> 用户："帮我找杭州地区营收5000万以上的制造业企业"
>
> 助理：（联网搜索后）已整理出34家潜在客户，预览如下...您可以下载完整 Excel，或说"全部加入漏斗"。

**可指定的筛选条件**：

- 城市/区域
- 行业
- 规模（人员数、营收）
- 资质标签（专精特新、规上企业等）

**v4.1 Agent-Reach全渠道增强**：安装 Agent-Reach 后，客户挖掘将自动使用 Exa AI 深度搜索、LinkedIn 企业信息、微信公众号行业报道、企业官网抓取等全渠道数据源，大幅提升搜索结果的丰富度和准确性。

### 二、客户信息管理（v1.0）

**查询客户**：
> "查看苏州云创科技的信息"
> "搜索姓名里有'云创'的客户"

**编辑信息**：
> "更新云创科技的联系人：IT总监李总，电话138xxxx"
> "设置云创科技的决策链：李总是技术评估者，影响力4，态度支持"

**监控客户**：
> "监控这家公司"（设置重点关注标记）

**合同到期提醒**：当客户的合同到期日不足6个月时，系统会自动提示"合同即将到期，可能是切入窗口期"。

### 三、销售漏斗推进与活动记录（v1.0）

**漏斗阶段说明**：

| 阶段 | 默认概率 | 说明 |
|------|----------|------|
| 线索 | 10% | 初步识别的潜在客户 |
| 初步接触 | 20% | 已进行首次沟通 |
| 需求确认 | 40% | 明确了客户需求 |
| 方案演示 | 60% | 已展示产品方案 |
| 商务谈判 | 80% | 进入价格/条款协商 |
| 赢单 | 100% | 成功签约 |
| 输单 | 0% | 未能成交 |

**快速记录活动（一句话模式）**：
> "今天给云创王总打了电话，他们用友U8太老了，约下周三演示"

助理会自动解析出：
- 活动类型：电话
- 内容摘要：了解到用友U8版本老旧
- 结果：约定下周三演示
- 建议操作：推进漏斗阶段，更新客户方案信息

**查看漏斗看板**：
> "我的漏斗概览"
> "本周待跟进的客户有哪些"

### 四、销售方法论应用（v1.0）

**生成提问清单**：
> "帮我用SPIN方法论准备云创科技的提问"

**切换方法论**：
> "设置方法论为 BANT"

**三种方法论对比**：

| 方法论 | 核心框架 | 适用场景 |
|--------|----------|----------|
| SPIN | 情境→难点→暗示→效益 | 挖掘深层需求 |
| BANT | 预算→权限→需求→时间 | 快速评估商机质量 |
| MEDDIC | 指标→买家→标准→流程→痛点→支持者 | 复杂大单分析 |

### 五、数据导出与备份（v1.0）

**导出数据**：
> "导出数据"

系统会将 customers、sales_funnel、activity_log 三张表导出为一个 Excel 文件（每表一个 Sheet），提供下载。

**备份提醒**：每周五系统自动提醒您备份数据。

### 六、智能拜访准备（v2.0）

**触发方式**：
> "准备拜访苏州云创科技"
> "云创科技拜访准备"

**生成内容**：

助理会自动生成一页纸简报，包含 5 个板块：

**1. 公司概况与动态**：基础信息（行业、规模、营收）和联网搜索获取的近期新闻动态。

**2. 当前方案与痛点**：基于已录入的 `current_solution` 字段推测可能的业务痛点。例如：
> "在用 用友U8 V13.0（2018年上线，已运行8年），推测痛点：多工厂协同困难，版本老旧无法满足新业务需求。"

**3. 决策链切入点**：展示已知的关键决策人及其态度，并建议接触策略。

**4. 结构化提问清单**：基于您选择的方法论（SPIN/BANT/MEDDIC）自动生成针对该客户的提问清单。

**5. 参考案例**：匹配至少一个同行业、同规模的成功案例作为参考。

**进阶用法**：
> "帮我写一封给云创科技的会议邀约邮件"

助理可基于拜访准备内容，生成会议议程邮件草稿，您可直接复制发送。

### 八、会议纪要智能处理（v2.0）

**触发方式**：会议结束后，直接输入会议内容或要点。

**示例对话**：
> 用户："今天和云创李总开了个技术交流会。李总说他们现在U8最大的问题是多工厂数据不通，每个月关账要7天。他们明年Q1有IT预算，大概50万左右。李总个人比较认可我们的集成方案，但老板王总还没表态。另外他们也在看金蝶的方案。"

**助理处理逻辑**：

| 识别信号 | 动作 |
|----------|------|
| "U8多工厂数据不通" | 更新 current_solution.notes |
| "明年Q1有IT预算，50万" | 更新 estimated_deal_value，记录预算信号 |
| "李总认可" | 更新 decision_chain 中李总态度为"支持" |
| "王总还没表态" | 更新或新增王总，态度标记"未知" |
| "在看金蝶" | 记入 key_extracted_info 竞品字段 |

**正向反馈**：当检测到积极信号时，助理会给出鼓励：
> "这是一个关键进展——李总已明确认可集成方案的技术能力，且客户已有明确预算规划，值得记录为里程碑。"

**输出**：结构化会议纪要（讨论要点 + 结论 + 待办事项），自动存入 activity_log。

### 十、商机健康度评分与提醒（v3.0）

**自动触发**：每日首次对话时，助理自动计算并报告。

**评分维度**：

| 维度 | 规则 | 分数影响 |
|------|------|----------|
| 跟进间隔 | 超过7天 / 超过14天 | -15 / -30 |
| 行动计划 | 无 next_action | -20 |
| 关系趋势 | relationship_score 下降 | -10 |
| 痛点匹配 | 客户有老旧系统或明确需求 | +15 |
| 预算信号 | 有预算信号 | +20 |

基础分为 70 分，范围 0-100。

**输出示例**：
> 商机健康度预警（3个客户需关注）：
> 1. 南京新材料（35分）：超过14天未跟进，无下一步计划。建议：安排一次电话了解客户最新状况。
> 2. 无锡精密（48分）：关系评分下降，竞品已介入。建议：安排技术交流巩固关系。

**待复活客户**：当 next_contact_date 过期>7天且健康分<40时，助理会单独标注：
> 待复活客户（2个）：
> - 苏州恒达：上次跟进在32天前，建议发送行业案例激活
> - "需要我帮您生成激活话术吗？"

**手动触发**：
> "查看商机健康度"
> "哪些客户需要尽快跟进"

### 十一、商机"阻力点"智能诊断（v3.0）

**触发方式**：
> "分析一下苏州云创科技为什么卡住"
> "云创科技卡在哪里了"

**分析维度**：

1. **活动停滞**：最近活动频率、间隔趋势
2. **决策链阻力**：是否存在高影响力的反对者
3. **竞品威胁**：是否有竞品被提及
4. **预算缺失**：是否缺乏预算信号
5. **方法论缺口**：基于您选定的方法论，哪些关键维度信息缺失

**输出示例**：
> **商机阻力诊断：苏州云创科技**
>
> **可能卡点：**
> 1. 技术评估者李总监态度为"反对"，影响力4星，最近会议未正面回应。
> 2. 竞品金蝶已介入，客户提及价格对比。
> 3. 预算尚未明确释放。
>
> **突破建议：**
> 1. 安排针对李总监痛点的专项技术交流，邀请架构师参与。
> 2. 准备与金蝶的差异化对比，突出集成成本优势。
> 3. 在下次沟通中尝试获取预算时间表信息。

### 十七、赢单/输单复盘与交接（v4.0 新增）

#### 赢单流程

当您将商机标记为"赢单"时，助理会主动询问：

> "恭喜赢单！是否需要生成客户交接清单？"

**交接清单内容**：
- 签约前承诺的功能清单与交付时间表
- 关键联系人的沟通偏好（如"李总喜欢看数据报表，讨厌听技术细节"）
- 实施过程中需特别注意的风险点
- 后续服务建议

**设置履约节点**：赢单后可录入关键节点日期：
> "设置履约节点：首付款5月1日，启动会5月15日，预计验收8月30日"

这些日期会出现在每日提醒中。

#### 输单流程

当您将商机标记为"输单"时，助理会主动追问两个问题：

> 1. "最终选了哪家？"（用友/SAP/金蝶/其他）
> 2. "主要原因是什么？"（价格/产品功能/品牌/关系/商务条款/交期/服务）

回答会以 JSON 格式存入日志，供后续分析：

```json
{"lost_to": "用友", "reason": "价格"}
```

**价值**：积累一段时间后，可通过"输单原因分析"查看趋势分布。

### 二十二、团队负责人专属功能（v4.0 新增）

以下功能仅在角色为"销售团队负责人"时激活。

#### 数据汇聚方式

1. 各销售定期导出个人数据为 Excel
2. 负责人上传所有 Excel 至助理
3. 助理自动合并、去重、生成团队看板

#### 18 项团队管理能力

**看板与统计**：

| 指令 | 说明 |
|------|------|
| "团队漏斗看板" | 整体阶段分布、按销售排行、健康度预警 |
| "团队业绩仪表盘" | 本月赢单金额、赢单率、客单价、环比 |
| "活动量审计" | 各销售新增客户、活动数、跟进间隔对比 |
| "销售预测分级" | 将漏斗分为 Commit/Best Case/Pipeline |

**预警与检测**：

| 指令 | 说明 |
|------|------|
| "撞单检测" | 模糊匹配重复跟进的客户，建议归属 |

| "续约/增购预警" | 赢单客户中合同将到期或长期无活动者 |
| "销售负荷监测" | 标注并行商机>15且周活动<5的销售 |

**分析与诊断**：

| 指令 | 说明 |

| "销售能力对标" | 五维度评分（获客/推进/关单/活动量/关系） |

**情报与协作**：

## 数据安全与隐私

1. **本地存储**：所有数据保存在本地 SQLite 数据库中，不会上传到云端
2. **定期备份**：建议每周导出 Excel 备份，系统每周五会提醒
3. **隐私合规**：助理仅使用您提供的信息或合法公开的企业信息
4. **数据可迁移**：通过 Excel 导出可迁移到其他 CRM 系统

## 注意事项

1. **数据确认**：所有数据修改操作前，助理会先让您确认再执行
2. **隐私保护**：仅使用您提供或合法公开的信息
3. **参谋定位**：所有建议均为参考意见，最终决策权在您手中
4. **数据存储**：数据保存在本地 SQLite 数据库中，请定期备份
5. **拜访简报质量取决于已录入的客户数据**：决策链、当前方案等字段越完善，简报越精准
6. **会议纪要处理后请确认**：助理会列出要更新的字段，请确认后再执行
7. **关键日期请注意核实**：助理从自然语言中提取日期可能有偏差，请确认后再依赖
8. **健康度评分基于已录入数据**：数据越完整，评分越准确
9. **阻力点诊断需要足够的活动记录**：建议至少有3条以上跟进记录后使用
10. **竞品话术仅供参考**：请结合实际情况和公司官方说法调整后使用
11. **业绩统计基于数据库记录**：确保赢单/输单状态及时更新以获得准确数据

> **v4.1 推荐**：强烈建议安装 Agent-Reach Skill 以获得最佳体验。Agent-Reach 覆盖 17 个平台渠道（搜索引擎、社交媒体、职业社交、开发者平台、网页与文章、视频平台），安装后客户挖掘、舆情监控和外部情报扫描功能将自动使用全渠道搜索能力，显著提升信息覆盖面和时效性。

## 常见问题

**Q：如何从旧版本升级？**
A：直接用新版文件覆盖旧版，运行 init_db.py 即可。已有数据完整保留。

**Q：团队负责人功能如何汇聚多人数据？**
A：各销售定期导出 Excel，负责人上传合并。未来可对接企业 CRM。

**Q：案例库如何建设？**
A：逐条说"添加案例"录入，或准备 CSV 批量导入。建议优先录入赢单客户。

**Q：如何切换角色？**
A：说"修改我的偏好"，重新选择角色即可。
