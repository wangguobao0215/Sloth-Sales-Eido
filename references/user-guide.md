# Sloth-Sales-Eido 用户使用指南

> 版本：4.1.0 | 适用于 Sloth-Sales-Eido v4.1.0（完整版）

---

## 一、技能定位

Sloth-Sales-Eido 是一款面向 B2B 销售人员和销售团队负责人的智能助理，覆盖从线索挖掘到赢单复盘的完整销售周期，内置 22+ 项核心能力。它不仅是一个 CRM 记录工具，更是一个主动型销售参谋，能够提供拜访准备、商机诊断、竞品应对、舆情监控、业绩分析等智能化支持。

**核心价值**：

- 一站式管理客户信息、销售漏斗和跟进记录
- 自然语言一句话快速记录活动，自动解析并建议后续操作
- 集成 Agent-Reach 全渠道搜索，实时获取客户动态和市场情报
- 支持 SPIN/BANT/MEDDIC 三种主流销售方法论
- 团队负责人可管理多人数据，进行团队分析和预警

**适用角色**：

| 角色 | 可用功能 |
|------|----------|
| 一线销售 | 全部个人功能（1-21 项） |
| 销售团队负责人 | 全部个人功能 + 18 项团队管理功能 |

## 二、安装与初始化

### 2.1 安装

将 `Sloth-Sales-Eido/` 整个目录复制到 QoderWork 技能目录下：

```bash
cp -r Sloth-Sales-Eido ~/.qoderwork/skills/
```

从旧版本升级时直接覆盖文件即可，运行初始化脚本会自动补建缺失表，已有数据完整保留。

### 2.2 依赖项

| 依赖 | 用途 | 安装方式 |
|------|------|----------|
| Python 3.8+ | 数据库操作、Excel 生成 | 系统预装或自行安装 |
| openpyxl | Excel 导出 | `pip install openpyxl` |
| 联网能力 | 客户挖掘、舆情扫描、政策检索 | QoderWork 内置 |
| Agent-Reach（可选，强烈推荐） | 17 平台全渠道搜索增强 | 安装 Agent-Reach Skill |

### 2.3 初始化数据库

首次使用时 Skill 会自动引导初始化，也可手动运行：

```bash
python ~/.qoderwork/skills/Sloth-Sales-Eido/scripts/init_db.py
```

数据库路径：`~/.qoderwork/data/Sloth-Sales-Eido/sales_crm.db`

### 2.4 首次使用引导

首次对话时，助理会自动发起引导流程，依次收集以下信息（每次只问 1-2 个问题）：

1. **角色确认** -- 一线销售 / 销售团队负责人
2. **产品类型** -- 软件/SaaS / 工业设备 / 专业服务 / 其他
3. **方法论偏好** -- SPIN（默认）/ BANT / MEDDIC
4. **数据导入** -- 上传 Excel/CSV 或从零开始

完成后偏好保存至 `user_preferences` 表。可随时说"修改我的偏好"进行调整。

### 3.2 核心 CRM 功能（v1.0，5 项）

**1. 客户挖掘与信息补全**

支持按区域、行业、规模等条件搜索潜在客户。集成 Agent-Reach 后可通过 Exa AI、Jina Reader、GitHub、LinkedIn、微信公众号等多渠道采集信息。输出表格预览并提供 Excel 下载。

**2. 客户信息管理**

增删改查客户记录，管理决策链（角色、影响力、态度），维护联系人。合同到期不足 6 个月自动提醒。

**3. 销售漏斗推进与活动记录**

标准阶段：线索 -> 初步接触 -> 需求确认 -> 方案演示 -> 商务谈判 -> 赢单/输单。支持一句话快速记录，自动解析活动类型、摘要、结果，建议推进操作。

**4. 销售方法论应用**

| 方法论 | 核心框架 | 适用场景 |
|--------|----------|----------|
| SPIN | 情境 -> 难点 -> 暗示 -> 效益 | 挖掘深层需求 |
| BANT | 预算 -> 权限 -> 需求 -> 时间 | 快速评估商机 |
| MEDDIC | 指标 -> 买家 -> 标准 -> 流程 -> 痛点 -> 支持者 | 复杂大单分析 |

**5. 数据导出与备份**

一键导出 customers、sales_funnel、activity_log 三张表为 Excel 文件。每周五自动提醒备份。

### 3.3 智能增强功能（v2.0，4 项）

**6. 智能拜访准备（Pre-Call Brief）**

说"准备拜访【客户名】"后，自动生成包含 5 个板块的一页纸简报：公司概况与动态、当前方案与痛点、决策链切入点、方法论提问清单、参考案例。还可进一步生成会议邀约邮件草稿。

**7. 个性化价值主张生成器**

说"生成价值主张【客户名】"后，基于客户痛点输出 3 条可直接使用的定制价值点。

**8. 会议纪要智能处理**

提供会议内容后，自动提取竞品、预算、时间节点等关键信息，更新数据库字段，生成结构化纪要存入活动日志。

**9. 关键日期与合规提醒**

自动识别关键日期（报价有效期、投标截止、合同到期等），每日首次对话展示"未来 7 天关键日期"。

### 3.4 分析与竞争功能（v3.0，7 项）

**10. 商机健康度评分与提醒**

自动计算 0-100 分健康评分，每日首次对话列出低于 50 分的客户。过期 >7 天且 <40 分的标注为"待复活客户"。

**11. 商机阻力点智能诊断**

说"分析一下【客户名】为什么卡住"，输出多维度卡点报告与突破建议。

**12. 竞品应对与话术生成**

按"认同 -> 差异化 -> 风险 -> 案例 -> 邀约"五步结构生成话术。集成 Agent-Reach 后可采集社交媒体上的竞品用户反馈作为实时论据。

**13. 协作快照生成**

200 字结构化摘要，便于内部转发同事。

**14. 内部求助话术生成**

一键生成可复制到 IM 群的 @同事求助话术。

**15. 决策人角色化沟通**

按经济决策者/技术评估者/使用者/采购执行者的关注点，自动调整邮件或话术风格。

**16. 个人业绩仪表盘与行为反馈**

展示新增线索、赢单金额、漏斗总额等核心指标及环比变化，附带轻量行为诊断建议。

### 3.5 完整版新增功能（v4.0，7 项）

**17. 赢单/输单复盘与交接**

赢单时生成客户交接清单并设置履约节点提醒；输单时自动追问竞品和原因，存入日志供后续分析。

**18. 客户动态舆情监控**

对标记为"监控"的客户进行全渠道新闻扫描（Exa AI、微信公众号、Twitter/微博、LinkedIn、Jina Reader），输出简报和行动建议。

**19. 标杆案例智能匹配**

基于客户画像匹配案例库中最相似案例，输出客户化场景描述。

**20. 行业政策/红利提醒**

联网检索当地产业扶持政策，作为拜访沟通的额外价值点。

**21. 案例库维护**

支持逐条引导录入和 CSV 批量导入两种方式。

**22. 团队负责人专属功能**

角色为"销售团队负责人"时激活 18 项团队管理能力，包括：

| 类别 | 功能 |
|------|------|
| 看板与统计 | 团队漏斗看板、团队业绩仪表盘、活动量审计、销售预测分级 |
| 预警与检测 | 撞单检测、停滞商机预警、续约/增购预警、销售负荷监测 |
| 分析与诊断 | 输单原因分析、流程瓶颈诊断、销售能力对标、售前资源 ROI 排序 |
| 情报与协作 | 外部情报扫描、自定义预警规则、销售反馈生成、团队周报、导出团队汇总表 |

## 四、典型工作流程

### 4.1 日常使用流程

1. **每日首次对话** -- 系统自动展示未来 7 天关键日期 + 待复活客户
2. **一句话记录活动** -- 如"今天给云创王总打了电话，约了下周演示"
3. **确认更新** -- 系统解析后建议更新漏斗阶段和跟进计划，确认即可

### 4.2 拜访准备流程

1. 说"准备拜访【客户名】"
2. 查看一页纸简报（概况、痛点、决策链、提问清单、案例）
3. 可选：说"生成会议邀约邮件"

### 4.3 商机推进流程

1. 查看漏斗概览，识别重点客户
2. 对卡住的商机说"分析一下 XX 为什么卡住"
3. 根据诊断结果制定突破策略
4. 若遇竞品，说"客户说 XX 更好，怎么回应？"
5. 需要内部支持时说"帮我申请售前支持 XX"

### 4.4 每周节奏

| 时间 | 系统自动输出 |
|------|-------------|
| 每周一上午 | 本周待跟进客户清单 + 健康度预警 |
| 每周五下午 | 本周战绩总结 + 行为反馈 |
| 每周五 | 数据备份提醒 |

## 五、最佳实践

1. **坚持一句话记录** -- 每次客户沟通后立即记录，保持数据鲜活。系统会自动解析并建议后续操作，无需手动分类。

2. **善用监控标记** -- 对重点客户说"监控这家公司"，让舆情扫描功能持续追踪客户动态，及时捕捉切入窗口。

3. **方法论匹配场景** -- 挖掘需求阶段使用 SPIN，快速筛选商机使用 BANT，大型复杂项目使用 MEDDIC。可随时切换。

4. **定期建设案例库** -- 每次赢单后说"添加案例"，积累标杆案例。案例越丰富，匹配和拜访准备的价值越大。

5. **关注健康度评分** -- 低于 50 分的商机需要立即关注。"待复活客户"是潜在的流失信号，应优先处理。

6. **安装 Agent-Reach** -- 强烈推荐安装 Agent-Reach Skill，可为客户挖掘、舆情监控、竞品应对等功能提供 17 平台全渠道数据支持。

## 六、常见问题（FAQ）

### Q1: 数据库初始化失败怎么办？

确保 Python 3.8+ 已安装，且路径 `~/.qoderwork/data/Sloth-Sales-Eido/` 目录有写入权限。可尝试手动创建该目录后重新运行 `init_db.py`。

### Q2: 如何从旧版本升级？

直接将新版文件覆盖到 `~/.qoderwork/skills/Sloth-Sales-Eido/` 目录下，然后运行 `python ~/.qoderwork/skills/Sloth-Sales-Eido/scripts/init_db.py`。脚本会自动补建缺失表，已有数据完整保留。

### Q3: 没有安装 Agent-Reach，功能会受影响吗？

不安装 Agent-Reach 时，客户挖掘、舆情监控等功能会退回基础联网搜索，仍然可用但信息覆盖面和深度会有所降低。强烈建议安装以获得最佳体验。

### Q4: 团队负责人如何汇聚多人数据？

各销售人员定期使用"导出数据"功能导出个人 Excel，团队负责人上传这些文件后，系统自动合并、去重并生成团队看板。

### Q5: 如何修改已设置的偏好（角色、方法论等）？

随时说"修改我的偏好"即可重新设置角色、产品类型和方法论等选项。
