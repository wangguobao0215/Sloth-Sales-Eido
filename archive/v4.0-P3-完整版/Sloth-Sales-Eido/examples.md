## 示例 3：拜访准备简报

```
用户：准备拜访苏州云创科技

助理：## 客户拜访简报：苏州云创科技
      日期：2026-04-16

      ### 1. 公司概况
      - 行业：制造业（电子元器件）
      - 规模：300-500人，年营收约8000万
      - 资质：专精特新企业
      - 近期动态：正在筹建第二工厂

      ### 2. 当前方案与痛点
      - 在用方案：用友U8 V13.0（2018年上线，已运行8年）
      - 推测痛点：多工厂协同困难，财务关账周期长

      ### 3. 决策链
      - 李总（IT总监）：技术选型主导，态度中立偏支持，关注系统稳定性
      - 建议切入：先争取李总技术认可，再推动上报

      ### 4. SPIN 提问清单
      - S（情境）："目前U8主要支撑哪些业务模块？"
      - P（难点）："新工厂投产后，U8能支撑多工厂协同吗？"
      - I（暗示）："如果关账周期继续延长，对季度决策会有什么影响？"
      - N（效益）："如果能把关账从7天缩短到2天，对您意味着什么？"

      ### 5. 参考案例
      同行业杭州精密电子（500人，原用U8），切换后关账周期缩短70%。

      需要我生成会议议程邮件草稿吗？
```

## 示例 4：竞品应对

```
用户：客户说SAP更稳定，怎么回应？

助理：针对苏州云创科技的情况，建议这样回应：

      1. 认同："您说得对，SAP确实是全球知名品牌，稳定性有口碑。"
      2. 差异化："不过SAP的优势主要在超大型企业场景，对于贵司300人的体量，
         实施周期通常需要12-18个月，定制开发成本可能超出预算30%以上。"
      3. 风险提示："SAP在国内中型制造企业的实施成功率约60%，主要卡在
         本地化需求响应和二次开发成本上。"
      4. 案例佐证："同行业杭州精密电子去年也对比过SAP，最终选择我们的原因是：
         实施周期仅4个月，且本地团队2小时响应。"
      5. 行动邀约："建议安排一次技术交流，让贵司IT团队实际操作对比。
         我们可以提供为期2周的POC验证。"
```
