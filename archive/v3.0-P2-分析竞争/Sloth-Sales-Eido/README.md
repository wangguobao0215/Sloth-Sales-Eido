# B2B 销售智能助理 v3.1（分析竞争版）

## 概述

在智能增强版基础上，增加商机健康度评分、阻力点智能诊断、竞品应对话术、协作快照、内部求助话术、决策人角色化沟通和个人业绩仪表盘七大分析与竞争能力，让销售从"记录型"进化为"分析型"。

## 功能清单

### 核心 CRM 功能（继承自 v1.0，5项）

1. **客户挖掘与信息补全** — 按区域/行业/规模搜索并补全客户信息（支持 Agent-Reach 多平台数据增强）
2. **客户信息管理** — 增删改查客户、决策链、联系人
3. **销售漏斗推进与活动记录** — 阶段管理、一句话快速记录
4. **销售方法论应用** — 支持 SPIN/BANT/MEDDIC 三种方法论
5. **数据导出与备份** — 一键导出 Excel

### 智能增强功能（继承自 v2.0，4项）

6. **智能拜访准备（Pre-Call Brief）** — 一页纸客户简报
7. **个性化价值主张生成器** — 基于客户痛点定制 3 条价值点
8. **会议纪要智能处理** — 自动提取关键信息并更新数据库
9. **关键日期与合规提醒** — 自动识别并追踪重要日期

### v3.0 新增功能（7项）

10. **商机健康度评分与提醒** — 多维度健康分计算 + 待复活客户标注
11. **商机"阻力点"智能诊断** — 综合分析卡点原因并给出突破建议
12. **竞品应对与话术生成** — 基于客户情况生成结构化应对话术（Agent-Reach社交媒体舆情增强）
13. **协作快照生成** — 200字摘要便于内部转发
14. **内部求助话术生成** — 一键生成 @同事的 IM 话术
15. **决策人角色化沟通** — 按角色关注点生成定制邮件/话术
16. **个人业绩仪表盘与行为反馈** — 战绩统计 + 行为诊断

## 安装

将 `Sloth-Sales-Eido/` 整个目录复制到 `~/.qoderwork/skills/` 下即可。

```bash
cp -r Sloth-Sales-Eido ~/.qoderwork/skills/
```

从 v1.0/v2.0 升级：直接覆盖文件，运行 `init_db.py` 会自动创建新增的 `case_library` 表，已有数据不受影响。

首次使用时，Skill 会自动引导初始化。也可手动运行：

```bash
python ~/.qoderwork/skills/Sloth-Sales-Eido/scripts/init_db.py
```

## 功能详细说明

### 一、客户挖掘与信息补全（v1.0）

按区域、行业、规模等条件搜索潜在客户，联网搜索后整理成结构化信息，支持批量加入漏斗。可指定的筛选条件包括城市/区域、行业、规模（人员数/营收）和资质标签（专精特新、规上企业等）。

### 二、客户信息管理（v1.0）

增删改查客户记录，管理决策链（角色、影响力、态度），维护联系人信息，设置重点关注标记。当客户合同到期日不足6个月时自动提示切入窗口期。

### 三、销售漏斗推进与活动记录（v1.0）

管理销售漏斗阶段（线索→初步接触→需求确认→方案演示→商务谈判→赢单/输单），支持一句话快速记录跟进活动，助理自动解析活动类型、内容摘要、结果和建议操作。

漏斗阶段与默认概率：

| 阶段 | 默认概率 | 说明 |
|------|----------|------|
| 线索 | 10% | 初步识别的潜在客户 |
| 初步接触 | 20% | 已进行首次沟通 |
| 需求确认 | 40% | 明确了客户需求 |
| 方案演示 | 60% | 已展示产品方案 |
| 商务谈判 | 80% | 进入价格/条款协商 |
| 赢单 | 100% | 成功签约 |
| 输单 | 0% | 未能成交 |

### 四、销售方法论应用（v1.0）

支持三种主流 B2B 销售方法论，可随时切换：

| 方法论 | 核心框架 | 适用场景 |
|--------|----------|----------|
| SPIN | 情境→难点→暗示→效益 | 挖掘深层需求 |
| BANT | 预算→权限→需求→时间 | 快速评估商机质量 |
| MEDDIC | 指标→买家→标准→流程→痛点→支持者 | 复杂大单分析 |

可自动基于所选方法论为特定客户生成结构化提问清单。

### 五、数据导出与备份（v1.0）

一键导出 customers、sales_funnel、activity_log 三张表为 Excel 文件（每表一个 Sheet）。每周五系统自动提醒备份。

### 六、智能拜访准备（v2.0）

触发词"准备拜访【客户名】"后，自动生成涵盖以下 5 个板块的一页纸简报：

1. **公司概况与动态**：基础信息 + 联网搜索近期新闻
2. **当前方案与痛点**：基于 current_solution 字段推测业务痛点
3. **决策链切入点**：关键决策人态度分析与接触策略建议
4. **结构化提问清单**：基于所选方法论自动生成针对性提问
5. **参考案例**：匹配同行业同规模的成功案例

支持进阶用法：基于拜访准备内容生成会议邀约邮件草稿。

### 七、个性化价值主张生成器（v2.0）

触发词"生成价值主张【客户名】"后，基于客户的当前方案痛点输出 3 条可直接使用的定制价值点。适用于邮件标题、方案摘要页、电话开场白和演示开场。

### 八、会议纪要智能处理（v2.0）

提供会议内容后，自动提取竞品、预算、时间节点等关键信息，更新数据库字段（current_solution、estimated_deal_value、decision_chain、key_extracted_info 等），并生成结构化纪要（讨论要点 + 结论 + 待办事项），自动存入 activity_log。检测到积极信号时会给出鼓励和里程碑建议。

### 九、关键日期与合规提醒（v2.0）

从跟进记录中自动识别关键日期（报价有效期、合同到期日、投标截止日、演示安排、付款节点），存储到 key_date 字段。每日首次对话时展示未来 7 天的关键日期清单。

### 十、商机健康度评分与提醒（v3.0 新增）

基于跟进间隔、行动计划、关系趋势、痛点匹配和预算信号五个维度自动计算 0-100 分，每日首次对话提醒低于 50 分的客户。

**评分维度**：

| 维度 | 规则 | 分数影响 |
|------|------|----------|
| 跟进间隔 | 超过7天 / 超过14天 | -15 / -30 |
| 行动计划 | 无 next_action | -20 |
| 关系趋势 | relationship_score 下降 | -10 |
| 痛点匹配 | 客户有老旧系统或明确需求 | +15 |
| 预算信号 | 有预算信号 | +20 |

基础分为 70 分，范围 0-100。当 next_contact_date 过期>7天且健康分<40时，标注为"待复活客户"。

### 十一、商机"阻力点"智能诊断（v3.0 新增）

综合活动日志、决策链反对者、竞品介入、预算缺失和方法论缺口五个维度，输出结构化的卡点报告和突破建议。触发词"分析一下【客户名】为什么卡住"。

### 十二、竞品应对与话术生成（v3.0 新增）

按"认同→差异化→风险→案例→邀约"五步结构，结合客户实际情况生成话术。触发词"客户说【竞品】更好，怎么回应？"。

**Agent-Reach 社交媒体舆情增强**：当安装了 Agent-Reach 技能时，系统会自动通过 Twitter、Reddit、V2EX 等社交媒体渠道搜索竞品的真实用户反馈和吐槽，将这些实时舆情数据作为话术中"风险提示"和"案例佐证"步骤的有力论据，使应对话术更具说服力。

### 十三、协作快照生成（v3.0 新增）

200字内的结构化客户摘要，涵盖基本信息、当前方案、核心痛点、关键人和当前阶段，可直接复制转发给售前顾问、技术专家或上级领导。

### 十四、内部求助话术生成（v3.0 新增）

一键生成可直接复制到企业 IM 的 @同事求助话术，包含客户背景、痛点、演示对象、时间安排和重点展示内容。

### 十五、决策人角色化沟通（v3.0 新增）

根据收件人角色自动调整内容重点和语言风格：

| 角色 | 关注重点 | 语言风格 |
|------|----------|----------|
| 经济决策者 | ROI、战略价值、风险控制 | 简洁数据驱动 |
| 技术评估者 | 性能、SLA、安全、集成 | 技术深度 |
| 使用者 | 易用性、培训、效率 | 场景化 |
| 采购执行者 | 价格、条款、资质 | 商务规范 |

### 十六、个人业绩仪表盘与行为反馈（v3.0 新增）

一句"我的业绩"即可看到新增线索、赢单金额、漏斗总额等核心指标及环比变化，并附带行为诊断（如"电话跟进中有70%未约定下一步具体行动"）。

## 特殊时间节点自动提醒

| 时间 | 自动输出内容 |
|------|-------------|
| 每日首次对话 | 未来7天关键日期 + 待复活客户 |
| 每周一上午 | 本周待跟进客户清单 + 健康度预警 |
| 每周五下午 | 本周战绩总结 + 行为反馈 |
| 客户合同到期前6个月 | 切入窗口期提醒 |
| 每周五 | 数据备份提醒 |

## 适用场景

- 已使用核心 CRM 和智能增强功能，需要深度分析能力
- 面对复杂商机需要诊断卡点原因
- 频繁遭遇竞品竞争，需要结构化应对话术
- 需要内部协作和跨角色沟通支持
- 希望量化个人销售表现并持续改进
